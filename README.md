# eink-chess

專為 Kobo e-ink 電子書閱讀器設計的單頁棋局，無框架、無依賴，只有一個 `index.html`。

## 使用方法

### 本機瀏覽

直接用瀏覽器開啟 `index.html`，不需要伺服器。

### Kobo 上使用

1. 將 `index.html` 複製到 Kobo 的儲存空間（任意目錄）
2. 用 Kobo 內建瀏覽器（Beta Browser）開啟該檔案
3. 或透過 GitHub Pages 部署後，用瀏覽器訪問網址

### GitHub Pages 部署

```bash
git push origin feature/ui-enhancements
# 在 GitHub 設定 Pages → 來源選擇對應分支
```

## 操作說明

| 操作 | 方法 |
|------|------|
| 選棋子 | 點擊己方棋子，合法落點會顯示提示圓點 |
| 移動 | 點擊目標格 |
| 取消選取 | 再次點擊已選棋子 |
| 升變 | 兵走到底線時自動彈出選擇面板（后/車/象/馬） |
| 悔棋 | 點「悔棋」，AI 模式下會同時撤回 AI 那步 |
| 新局 | 點「新局」重置棋盤 |
| 切換 AI | 點「AI: 開/關」 |
| 調整難度 | 點「難度 1/2/3」循環切換 |

白方為玩家，黑方為 AI（預設開啟）。

## 功能

- **合法走步提示** — 選棋子後顯示可落點（實心點）及可吃子格（邊框）
- **最後一步標示** — 走完後該步起點與終點顯示外框
- **升變選擇** — 兵升變時彈出面板，可選后、車、象、馬
- **走步記錄** — 畫面下方即時顯示代數記號（含 `+` 將軍、`#` 將死、`O-O` 易位）
- **棋盤座標** — 左側顯示 1–8，底部顯示 a–h
- **將死 / 和局偵測** — 無合法走步時自動判定結果
- **AI 對弈** — minimax + alpha-beta 剪枝，三段難度

## 技術

- **單一檔案** — 全部 HTML / CSS / JavaScript 寫在 `index.html`，約 350 行
- **無外部依賴** — 不使用任何函式庫或框架
- **棋盤表示** — 64 格一維陣列，正數為白子、負數為黑子
- **走步生成** — `pseudoMoves()` 產生偽合法步，`legalMoves()` 過濾留王將軍的走法
- **AI** — negamax 架構的 minimax，alpha-beta 剪枝，深度 1–3 可調
- **評估函數** — 棋子基礎值 + 位置獎勵表（Pawn / Knight / Bishop / King）
- **特殊走法** — 王車易位、吃過路兵（en passant）、兵升變
- **RWD 棋盤** — 啟動時依視窗大小動態計算格子尺寸
- **e-ink 最佳化** — 字體與按鈕放大 3 倍，純黑白配色，無動畫

## 棋子符號對照

| 符號 | 白方 | 黑方 |
|------|------|------|
| P / p | 兵 | 卒 |
| N / n | 馬 | 馬 |
| B / b | 象 | 象 |
| R / r | 車 | 車 |
| Q / q | 后 | 后 |
| K / k | 王 | 王 |

白方棋子以灰色顯示（`#555`），黑方棋子以黑色顯示（`#000`）。
